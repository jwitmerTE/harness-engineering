Harness Engineering · System Overview

A mini agent runtime that makes an
LLM loop survive production.

A bare LLM loop fails in predictable ways: a crash loses everything, a tool fires twice, context grows until the model degrades, an approval stalls for days. The harness handles all of it. The domain is ordinary support triage — the runtime is the point.

Agent systems are workflow systems. The LLM decides the next semantic step. The harness owns execution.

01

The execution loop

Everything in the codebase contributes to one durable, resumable loop. Each step maps to a module. All non-determinism lives inside durable steps, so the loop body can be replayed deterministically after a crash.

02

How the pieces connect

The harness and the inspector never share business logic. They share one thing: a typed event stream. Every action becomes an AgentEvent, persisted before it is broadcast.

04

Tools split by consequence

The tool set is tiered. The tier decides whether a call is safe to re-run after a crash, whether it is a durable step, and whether it needs a human.

Read / compute

  • runCode
  • classifyItem
  • draftReply

Safe to re-run after a crash. No side effects to duplicate.

Side-effecting

  • sendReply
  • issueRefund · needs approval

Each call is a durable step. issueRefund moves money — it also suspends for human approval.

Routing

  • handoff

Intercepted by the runtime — never reaches runTool. It swaps the active agent and continues the conversation.

05

Key dependencies

Two libraries earn their place. Everything else is built from scratch and the production tools are named, not depended on.

Durable execution

@dbos-inc/dbos-sdk

Wrapping a function in DBOS.runStep() checkpoints its result to Postgres before the workflow continues. On crash, DBOS.launch() resumes every in-flight workflow from its last completed step — no repeated LLM calls, no duplicate side effects. recv/send extend the guarantee to human-approval suspend/resume.

In production: Temporal, Inngest.

Typed SQL

drizzle-orm

A type-safe query builder, used for the one event_log table whose data column is typed as AgentEvent. Every row read back is already the right shape — no runtime casting. Raw SQL would work; Drizzle earns its place through that single typed column.

Boundary: node:vm is not a true security boundary → e2b, Firecracker, Cloudflare Sandbox.
